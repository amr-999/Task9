A Transaction Log (T-Log) Backup is a specialized type of database backup that captures every single modification made to your database (INSERTs, UPDATEs, DELETEs, and structural changes) since the last log backup.While a Full Backup is a snapshot of the entire database at one moment, a Transaction Log Backup is like a "video recording" of everything that happened afterward.1. Why do you need it?Point-in-Time Recovery: This is the "Time Machine" of the database world. If a user accidentally deletes a table at 10:05 AM, you can restore your Full Backup and then "replay" the transaction logs up to 10:04 AM to get the data back.Preventing Log Growth: In databases using the Full Recovery Model (like SQL Server), the transaction log file will grow forever until you back it up. A log backup "truncates" the log, which marks the space inside the file as reusable for new transactions.Minimal Data Loss: You can run log backups very frequently (e.g., every 5 or 15 minutes). If your server crashes, you only lose the few minutes of data since the last log backup, rather than an entire day's worth of work.2. The "Log Chain"To restore a database using log backups, you must have an unbroken chain:Full Backup (The Foundation)Log Backup 1Log Backup 2Log Backup 3 ... and so on.[!CAUTION]If you lose even one log backup in the middle of the sequence, the "chain" is broken. You will not be able to restore any subsequent log backups beyond that gap.4. Basic SQL CommandTo take a log backup in SQL Server, you would use:SQLBACKUP LOG [YourDatabaseName]
